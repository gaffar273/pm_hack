1. The Problem
The Scene
A doctor opens a patient file. It contains:
  - 200 previous lab results spanning 5 years
  - 12 radiology reports
  - 8 specialist letters
  - 15 medications
  - 7 diagnoses from 4 different doctors
  - A new biopsy report that just came in 10 minutes ago
The doctor has 7 minutes with this patient.
They will read the new result. They will not read everything else.
They will miss context. Not because they are a bad doctor.
Because they are human, and the system gives them no choice.
The core problem is not the doctor. It is the absence of a system that connects a new result to everything already known about the patient and tells the doctor what it means in that specific context.
A sodium level of 128 means something completely different in:
A healthy 25-year-old athlete
A 70-year-old on diuretics with heart failure
A cancer patient currently on chemotherapy
Every test result needs to be read in the context of the whole patient. Right now, that context assembly happens inside the doctor's head, manually, under time pressure, with incomplete recall.
2. What ContextMD Does
Core Concept
Any new test result arrives ù blood test, biopsy, MRI report, ECG, pathology, surgical report.
ContextMD reads it, pulls the patient's entire FHIR history, cross-references medical literature,
and hands the doctor one intelligent briefing ù before they even open the file.

2.3 Clinical Scenarios Covered
Scenario
What ContextMD Does
Cancer diagnosis (new)
Reads pathology report, pulls staging indicators from imaging, cross-references oncology treatment guidelines (NCCN, ESMO), flags clinical trials, checks current meds for contraindications with chemotherapy agents
Cancer ù follow-up scan
Compares new scan to all previous imaging, identifies progression or remission trend, updates treatment recommendation based on response data
Pre-surgical assessment
Reads all relevant labs and imaging, checks anaesthesia risk factors (cardiac, renal, respiratory), flags anything that makes surgery high risk, produces pre-op summary for surgeon
Abnormal blood test
Compares to patient's full longitudinal lab history, identifies whether this is a new development or a worsening trend, checks medications that could cause this abnormality
Chronic disease monitoring
Diabetes HbA1c trend, kidney function trajectory, liver enzymes over time ù connects pattern to current treatment and recommends adjustment
Post-operative review
Reads surgical report and post-op labs together, flags early warning signs of complications, cross-references recovery guidelines for the specific procedure
Minor / routine result
Short clear summary ù result is within expected range given patient profile, no action required, or simple standard next step

3.3 SHARP Context Propagation
Every agent call and MCP tool call in the chain carries the SHARP context header ù patient ID, FHIR base URL, and access token. This means:
No agent re-authenticates separately
Patient scope never drifts between agent hops
MCP tools are generic ù they work for any patient, any hospital, any FHIR server
// SHARP header present on every MCP tool call
const sharp = JSON.parse(request.headers['x-sharp-context']);
// { fhir_base_url, patient_id, access_token, encounter_id }
const res = await fetch(`${sharp.fhir_base_url}/DiagnosticReport?patient=${sharp.patient_id}`, {
  headers: { Authorization: `Bearer ${sharp.access_token}` }
});

5.2 Clinical Reasoning Agent
Role
This is the core intelligence of the system. It takes the full context object and reasons about what the new result means for this specific patient.
Reasoning Steps
Interpret the result ù what does this finding mean clinically
Contextualise against history ù is this new, worsening, improving, or expected
Contextualise against conditions ù how does this interact with existing diagnoses
Build differential ù what conditions are consistent with this result given the full picture
Assess risk level ù low / moderate / high / critical ù with explicit reasoning
Propose next steps ù what should happen next, ordered by priority
Flag omissions ù what has NOT been done that guidelines say should be done
Risk Levels
Risk Level
Meaning
Example
Critical
Immediate action required
Troponin spike in patient with chest pain and prior MI history ù emergency referral
High
Action required within 24-48 hours
New cancer diagnosis ù urgent oncology referral needed
Moderate
Action required at next appointment
Rising HbA1c trend ù medication adjustment needed
Low
Monitoring only
Slightly low vitamin D ù supplement recommendation

5.4 Literature Synthesiser Agent
Role
Takes the raw literature retrieved by the MCP tool and filters it down to what is genuinely relevant to THIS patient. Not all guidelines apply to every patient.
Filtering Logic
Match guideline condition to patient's active diagnoses
Match patient age and sex to guideline eligibility criteria
Match result type to guideline scope ù e.g. post-surgery guidelines only if procedure recorded
Match clinical trial eligibility criteria against patient profile
Rank by recency ù prefer guidelines updated within last 3 years
Output
{
  "guidelines": [
    {
      "source": "NCCN Breast Cancer Guidelines v2.2025",
      "relevance": "Patient has ER+/HER2- invasive ductal carcinoma ù matches guideline scope exactly",
      "key_recommendation": "Adjuvant endocrine therapy recommended for 5-10 years",
      "url": "https://nccn.org/..."
    }
  ],
  "trials": [
    {
      "title": "monarchE ù Abemaciclib in early HR+ breast cancer",
      "status": "Recruiting",
      "patient_match": "ER+, node-positive ù patient meets primary criteria",
      "nct_id": "NCT03155997"
    }
  ]
}
